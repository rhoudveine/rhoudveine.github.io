01. Philosophy

"POSIX is for people who like constraints."

Custom Interfaces

We define our own syscalls and userlib interfaces that make sense for this specific kernel, prioritizing clean logic over legacy support.

Modern First

Native focus on ACPI, PCI-E, and UEFI-aware features. We aren't building for 386 hardware; we're building for today.

Engineered Components

[ VFS ]

Virtual File System

Robust abstraction supporting FAT32 (Read), RamFS, and ProcFS interfaces.

[ DRV ]

Hardware Stack

AHCI (SATA), xHCI (USB 3.0), and high-res framebuffer with double buffering.

[ VRY ]

VRAY Subsystem

PCI enumeration using binary search on a 20,000+ device database.

The Architecture Stack

VRAY (Bus Layer - PCI/PCI-E Enumeration)
↓
VNODE (Controller Layer - xHCI / AHCI)
↓
NVNODE (Peripheral Layer - USB HID / Disk)

Getting Started

Environment

  • • x86_64-elf-gcc
  • • nasm (Netwide Assembler)
  • • xorriso / mtools / grub

Boot Command

qemu-system-x86_64 -bios /usr/share/ovmf/OVMF.fd -cdrom dist/x86_64/kernel.iso -serial stdio -m 2G -smp 4

$ git clone https://github.com/Reyhank45/rhoudveine.git

$ cd rhoudveine

$ make ISO

[CC] kernel/main.c

[AS] arch/x86_64/boot.asm

[LD] dist/x86_64/kernel.bin

[OK] ISO created at /ISO/rhoudveine.iso
